# التحليل الكامل للنظام — منصة التأهيل والتوظيف

## 1. الرؤية
منصة وسيطة موثوقة تديرها **جمعية مستودع المدينة المنورة الخيري**، تربط المستفيدين بفرص
التدريب والتأهيل والتوظيف المقدّمة من الشركات والمعاهد وشركات التوظيف والجهات المانحة.

## 2. أصحاب المصلحة (Actors) والصلاحيات
| الدور | الوصف | أبرز الصلاحيات |
|------|-------|----------------|
| مدير النظام (ADMIN) | تحكّم كامل | إدارة المستخدمين، اعتماد المنظمات/الفرص، التقارير |
| موظف الجمعية (STAFF) | إدارة الحالات | اعتماد المستفيدين، إصدار الخطابات، المتابعة، تشغيل المطابقة |
| المستفيد (BENEFICIARY) | باحث عن فرصة | الملف الشخصي، رفع الوثائق، التقديم، الإشعارات |
| الشركة (COMPANY_ADMIN/HR) | جهة توظيف | إنشاء الوظائف، استعراض المرشحين، المقابلات، العروض |
| المعهد (INSTITUTE) | تدريب | البرامج والمقاعد والشهادات والتبرعات التدريبية |
| شركة التوظيف (RECRUITER) | وساطة توظيف | فرص التوظيف وإدارة المرشحين |
| الجهة المانحة (DONOR) | تبرعات | تبرع وظيفي/تدريبي/توظيفي ومتابعته |

> صلاحيات موظفي الشركة متعددة المستويات (موارد بشرية/مدير توظيف/مدير فرع/مشرف)
> تُدار عبر `SEC_USER_ROLES.scope_org` لعزل الدور داخل المنظمة.

## 3. الكيانات الأساسية (Domain)
- **المستخدمون** `RE_USERS` (موحّد لكل الأنواع) + RBAC (`SEC_*`).
- **المنظمات** `RE_ORGANIZATIONS` (شركة/معهد/شركة توظيف/مانح) مع اعتماد.
- **المستفيدون** `RE_BENEFICIARIES` + تصنيفات + مهارات + وثائق.
- **الفرص**: وظائف `RE_JOBS`، برامج تدريبية `RE_TRAINING_PROGRAMS`.
- **التبرعات** `RE_DONATIONS`: وظيفي / تدريبي / توظيفي بوحدات قابلة للاستهلاك.
- **الطلبات** `RE_APPLICATIONS` (وظيفة/تدريب) مع سجل أحداث ومقابلات وعروض.
- **المطابقة** `RE_MATCHES` (درجة 0–100 مخزّنة).
- **الخطابات** `RE_LETTERS` (تعريف/ترشيح/توصية) برقم مرجعي ورمز QR.
- **الإشعارات** `RE_NOTIFICATIONS` متعددة القنوات.

## 4. أنواع التبرع
1. **تبرع وظيفي**: شركة تتبرع بعدد وظائف لمسمى معيّن (مثال: 10 محاسب). تُربط الوظيفة
   بالتبرع وتُستهلك وحدة عند كل توظيف ناجح (`units_consumed`).
2. **تبرع تدريبي**: معهد يقدّم مقاعد مجانية أو خصماً (50%/100%) أو منحة، يُربط ببرنامج.
3. **تبرع توظيفي**: شركة توظيف توفّر عدداً من الفرص أو ترعى عملية التوظيف.

## 5. محرك المطابقة الذكي
درجة من 0 إلى 100 بأوزان: التخصص 30، المهارات 30، المؤهل 15، الخبرة 10، المدينة 10،
الرغبات 5. (التفاصيل في `db/08_pkg_matching.sql`). يُعاد الاحتساب عند:
تحديث ملف المستفيد، نشر وظيفة، أو طلب يدوي من الموظف.

## 6. تدفّقات رئيسية (Workflows)
- **اعتماد مستفيد**: تسجيل → استكمال الملف (نسبة اكتمال) → SUBMITTED → اعتماد الموظف.
- **نشر وظيفة**: إنشاء الشركة → PENDING → اعتماد المدير → PUBLISHED → استقبال الطلبات.
- **مسار الطلب**: SUBMITTED → SCREENING → SHORTLISTED → INTERVIEW → OFFER → HIRED.
- **إصدار خطاب**: الموظف يصدر خطاباً → رقم مرجعي + QR → تحقق عام عبر `/letters/verify`.

## 7. متطلّبات غير وظيفية
- أمان: JWT + RBAC + تجزئة كلمات المرور + سجل تدقيق + إبطال التوكنات.
- أداء: فهارس على الأعمدة الحرجة، تخزين نتائج المطابقة، صفحات ORDS.
- توسّع: فصل الواجهة (Vercel) عن الـ API (OCI)، طبقات حزم PL/SQL مستقلة.
- تجربة: عربية RTL، تصميم متجاوب، دعم PWA، إشعارات متعددة القنوات.
